PHP and MySQL Database Design

This page explains the database design and PHP structure used by The Computer Store. The website uses PHP for server-side functionality and MySQL for storing products, categories, users, orders, site settings, and system monitoring data.

The main PHP configuration file is backend/config.php. This file handles the database connection, HTML escaping helper functions, session startup, login state, admin checks, theme selection, redirect safety, and shared reusable functionality used by multiple pages across the website.

The main database file is backend/database.sql. It creates the database named store_db and defines the tables required by the site. The database is imported automatically by backend/install.php when the project is run on a new local setup.

  1. categories
    Stores product categories such as Storage, Audio, Input Devices, Cables, Power, and Accessories. Each category has an ID, name, slug, description, and creation date.
  2. products
    Stores the main catalogue records. Each product belongs to a category and includes the title, brand, description, base price, stock amount, image filename, featured flag, active flag, and creation date.
  3. product_options
    Stores selectable options for each product. Examples include color, storage, connectivity, length, and wattage. Each option is connected to one product and can also include an optional image and a price difference.
  4. users
    Stores registered accounts. Each user has a username, email, hashed password, full name, role, active status, and account creation date. Roles are either customer or admin.
  5. orders
    Stores order records placed by users. Each order includes the related user, total price, order status, and creation date.
  6. order_items
    Stores the individual items that belong to an order. Each record connects an order to a product and stores quantity, unit price, and selected options.
  7. site_settings
    Stores website-wide settings such as the currently active theme, site name, and maintenance mode.
  8. service_log
    Stores monitoring results from the backend monitor page. Each record stores the service name, status, response time, detail text, and check time.

The database uses foreign key relationships to keep records connected correctly. For example, each product belongs to one category, each product option belongs to one product, and each order item belongs to both an order and a product.

In the PHP files, prepared statements are used for most database queries. This helps protect the website from SQL injection and keeps the SQL easier to manage. The admin pages use PHP forms to insert and update product records, manage user account status, and change the site template stored in the database.

Overall, the PHP and MySQL design is organized so that the catalogue can be updated through the admin interface while keeping the public pages dynamic and connected to the database.
