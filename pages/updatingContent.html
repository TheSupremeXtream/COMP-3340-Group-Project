How to Update Site Content

This page explains how to update the website content in a simple way. These steps are written for someone who may not be a programmer and needs to change product images, multimedia files, or text content.

  1. Updating product records
    Log in as an administrator and open the Products page in the admin area. You can add a new product or edit an existing one using the product form.
  2. Updating the main product image
    First place the new image file inside the backend/images folder. After that, open the product in the admin form and type the image filename exactly as it appears in that folder. Example: Gaming_Mouse.jpg.
  3. Updating option images
    Some option records, such as color choices, can use their own image files. These image files should also be stored in the backend/images folder. The option image filename should match the file stored in that folder.
  4. Adding new catalogue images
    Save the image into backend/images. Use a short and clear filename with no spaces when possible. Good examples are Red_Controller.png or USB-C_Hub.jpg.
  5. Updating wiki screenshots and page images
    Images used by the help wiki pages are stored in the top-level images folder. Replace the old screenshot with a new one using the same filename, or update the filename inside the corresponding HTML page if needed.
  6. Updating the home page video
    The main store video is stored in the multimedia folder. The current home page uses the file promo.mp4. To replace the video, either keep the new file named promo.mp4 or update the filename inside index.php.
  7. Updating audio files
    Audio files such as bell.mp3 and whoosh.mp3 are stored in the multimedia folder. If you replace them using the same filename, the website will continue to use them automatically.
  8. Updating page text
    Static information pages such as the wiki pages are stored in the pages folder. Open the matching HTML or PHP file and change the visible text between the tags.
  9. Keeping file names consistent
    Make sure the filename written in the code matches the real file exactly, including capitalization and file extension such as .jpg, .png, .webp, .mp3, or .mp4.
  10. Final check after updating content
    After replacing files or editing page text, refresh the website and visit the related page to make sure the image, video, audio, or product content appears correctly.

In general, the easiest way to update content is to keep the same filename and replace the old file with the new one in the correct folder. This avoids needing extra code changes.
